# Milestone v1.2: skill-and-tool-guidance-hardening

**Status:** ✅ SHIPPED 2026-03-22
**Phases:** 19-21
**Total Plans:** 5

## Overview

This milestone made the standalone V3 surface executable from public guidance
alone. It turned direct-tool metadata, stage-skill contracts, and the README
into one coherent operator surface so agents can start, inspect, and continue
the standalone loop without reading source code.

## Phases

### Phase 19: Tool Catalog Operator Guidance

**Goal**: Developers can inspect a direct V3 CLI tool and know the common-path
request shape, the correct routing layer, and the next action after a
successful result.
**Depends on**: Phase 18
**Plans**: 2 plans

Plans:

- [x] 19-01-PLAN.md — Add structured common-path examples and explicit routing boundaries to the direct tool catalog payload.
- [x] 19-02-PLAN.md — Add structured follow-up semantics and regression coverage for next-step operator guidance.

**Details:**
The direct V3 tool catalog now exposes machine-readable examples, routing
boundaries, and follow-up semantics through the public `rdagent-v3-tool`
surface. This phase established the tool layer as an executable guidance
surface rather than a schema-only reference.

### Phase 20: Stage Skill Execution Contracts

**Goal**: Developers can start and continue the standalone stage loop from the
skill packages using explicit minimal inputs and default stop/continue
semantics.
**Depends on**: Phase 19
**Plans**: 2 plans

Plans:

- [x] 20-01-PLAN.md — Add explicit `rd-agent` start-contract and gated pause guidance.
- [x] 20-02-PLAN.md — Add continuation contracts and regression coverage for `rd-propose`, `rd-code`, `rd-execute`, and `rd-evaluate`.

**Details:**
The public skill surface now exposes the exact minimum `rd-agent` start
contract, the default gated pause semantics, and the paused-run continuation
contracts for all four stage skills. Missing-field recovery was explicitly
framed as agent-led instead of user-led tool browsing.

### Phase 21: Executable Public Surface Narrative

**Goal**: Developers can use the README and regression suite as the stable
public reference for the standalone V3 pipeline start, inspect, and continue
flows.
**Depends on**: Phase 20
**Plans**: 1 plan

Plans:

- [x] 21-01-PLAN.md — Recast `README.md` into a `Start -> Inspect -> Continue` public playbook and lock it with focused Phase 21 README regressions.

**Details:**
The README now acts as an agent-first `Start -> Inspect -> Continue` playbook
that points to the real skill and tool surfaces for detailed contracts while
preserving the older public section anchors and the public/internal
documentation boundary.

## Milestone Summary

**Key Decisions:**
- Keep the public V3 surface skill-first: `rd-agent` starts the main path,
  stage skills own paused-run continuation, and direct tools remain the
  selective downshift layer.
- Put operator guidance where the operator already looks: tool metadata,
  skill packages, and README, instead of creating another docs-only catalog.
- Keep README at the decision layer and link to the real `SKILL.md` files and
  `rdagent-v3-tool describe ...` output for field-level truth.

**Issues Resolved:**
- Closed the original schema-only catalog gap by giving direct tools concrete
  examples, routing guidance, and explicit next-step semantics.
- Removed ambiguity around how to start `rd-agent`, when the default path
  pauses, and which fields are required to continue each stage.
- Replaced the old section-oriented README narrative with one executable public
  path for start, inspect, and continue.

**Issues Deferred:**
- No dedicated `v1.2` milestone audit was run before archival.
- Broader branch-memory and exploration operator playbooks remain future work.

**Technical Debt Incurred:**
- `tests/test_phase14_stage_skills.py` still has an out-of-scope expectation
  mismatch against `.importlinter`; this was documented during Phase 20 but not
  fixed inside the milestone.
- Phase directories remain in `.planning/phases/` as execution history because
  this archive path kept the non-destructive default and did not move them into
  a separate milestone-only phase archive.

**Key Accomplishments:**
- Added concrete examples, routing guidance, and follow-up semantics to the
  public direct-tool catalog.
- Added explicit `rd-agent` start guidance and stage-skill continuation
  contracts to the public skill surface.
- Locked the public README narrative as an executable `Start -> Inspect ->
  Continue` playbook.
