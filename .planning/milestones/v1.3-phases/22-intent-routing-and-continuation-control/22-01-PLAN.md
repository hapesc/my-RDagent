<goal>
Users should be able to state their intent in plain language and be routed to
the right high-level action, with paused work taking precedence over silent new
run creation.
</goal>

<assumptions_constraints>

  later preflight/runtime blocker layer here; that belongs to Phase 23.
- Preserve the existing high-level skill hierarchy: `rd-agent` remains the
  public top-level path, and `rd-tool-catalog` remains a downshift surface.
- Preserve current skill contracts; this phase can update how entry routing
  chooses among them, not redefine all stage contracts.
- The user-facing response should default to current state, reason, and exact
  next action rather than long orchestration narration.
</assumptions_constraints>

<edge_cases>
- Multiple runs may exist; the router should not silently pick an arbitrary run
  without surfacing why it chose that continuation target.
- A paused run may exist but still require later preflight checks; Phase 22
  should route to the correct skill without pretending the work is executable
  yet.
- Plain-language intent must not promote `rd-tool-catalog` to a coequal start
  path.
</edge_cases>
