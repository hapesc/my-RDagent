<goal>
The default route-level "what next?" answer should sound like an operator
assistant, keep canonical truth visible, and give the user a minimum executable
next move instead of raw state-machine narration.
</goal>

<assumptions_constraints>
- Do not compute a second readiness or state truth path in this plan. Reuse the
  existing Phase 22 route logic and Phase 23 preflight fields.
- The new shared guidance layer is the only runtime truth source for overlapping
  guidance fields and stage-to-next-skill mapping. `rd_agent` must project it;
  it must not hand-assemble duplicate guidance fields or keep a second mapping.
- Preserve the existing top-level route field family:
  `current_state`, `routing_reason`, `exact_next_action`,
  `recommended_next_skill`, `current_action_status`,
  `current_blocker_category`, `current_blocker_reason`, and `repair_action`.
- Minimum start and continuation skeletons must reuse the exact public field
  names already locked in the `SKILL.md` contracts; do not invent aliases.
- A healthy paused run must not auto-dump a continuation skeleton. This plan
  may add a detail hint, but it must keep the default answer terse.
- The shared guidance helper and renderer must not read `StateStorePort` or call
  any `load_*` state helpers. They only format already-loaded route, preflight,
  and snapshot data passed in by the caller.
- Do not rewrite stage entrypoint messages or docs in this plan; that belongs
  to Plan 02 after the shared route-surface contract is stable.
</assumptions_constraints>
