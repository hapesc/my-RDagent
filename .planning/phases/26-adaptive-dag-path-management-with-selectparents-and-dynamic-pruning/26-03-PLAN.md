    Rewrite `v3/algorithms/prune.py` to extend `prune_branch_candidates` with new
    keyword-only parameters while keeping the existing signature fully backward-compatible.

    The extended function signature:
    ```python
    def prune_branch_candidates(
        candidates: list[tuple[str, float]],
        *,
        score_threshold: float | None = None,
        relative_threshold: float | None = 0.5,
        # Phase 26 extensions
        generalization_gaps: dict[str, float] | None = None,
        overfitting_risks: dict[str, float] | None = None,
        budget_ratio: float | None = None,
        min_active_branches: int = 1,
    ) -> list[str]:
    ```

    Logic changes:
    1. When `budget_ratio` is provided, compute `dynamic_threshold = cosine_decay(budget_ratio, high=0.7, low=0.3)`.
       Use this instead of `relative_threshold` for the cutoff calculation:
       `cutoff = max(cutoff, best_score * dynamic_threshold)`.
       When `budget_ratio` is None, fall back to existing `relative_threshold` behavior.

    2. After computing the cutoff, for each branch below cutoff:
       - If `generalization_gaps` is provided and branch's gap < 0.1, protect it
         (skip pruning -- low gap means good generalization despite lower score).
       - If `overfitting_risks` is provided and branch's risk > 0.6, prioritize it
         for pruning (add to front of prune list).

    3. Replace the final safety check: instead of `remaining < 1`, use
       `remaining < min_active_branches`. This respects the new min_active_branches
       parameter (default 1 preserves old behavior, Phase 26 uses 2).

    4. Keep the best_ids protection: the best-scoring branch(es) are never pruned.

    Import: `from v3.algorithms.decay import cosine_decay`

    Create `tests/test_phase26_pruning.py` with tests from behavior block.
