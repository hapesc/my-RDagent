## Part 3: Internal Workflows to Create

List each new workflow file with its scope boundary:

1. `skills/rd-agent/workflows/intent-routing.md` -- Owns: state inspection, paused-run detection, routing reply format, preflight gate interaction. Does NOT own: stage-level continuation, tool selection.
2. `skills/rd-agent/workflows/start-contract.md` -- Owns: minimum vs recommended payload, example shape, field validation order. Does NOT own: stage execution, branch lifecycle.
3. `skills/_shared/references/stage-continuation-contract.md` -- Owns: shared continuation fields (run_id, branch_id, summary, artifact_ids), recovery-from-missing protocol, escalation-to-tool-catalog protocol. Parameterized by stage name, next skill, and stage-specific extra fields.
4. `skills/_shared/references/tool-catalog-routing.md` -- Owns: when to downshift from any stage skill to rd-tool-catalog, the "keep operator on stage path" rule.
5. `skills/_shared/references/stage-failure-handling.md` -- Owns: missing-field inspection, state-first recovery, route-to-correct-stage logic. Parameterized by stage-specific failure paths.
